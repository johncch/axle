# Migrating to Axle 0.26.0

0.26.0 adds experimental context compaction and turns tool-loop limits into
stops instead of errors:

- `agent.onCompaction(callback)` + `await agent.compact()` replace the active
  conversation with a host-written shorter one (experimental)
- `agent.history` exposes `messages` (active), `archive` (raw append-only),
  and `compactions` (receipts)
- `maxContextTokens` bounds the tool loop by token budget; `maxIterations` and
  `maxContextTokens` now return `ok: true` with a `stopped` reason
- `TurnPart` gains a `compaction` member; compaction renders as an agent turn
  containing one such part

It also contains several breaking API changes. Applications that read
`agent.history.log`, call `agent.snapshot()` or `agent.restore()`, or key
stream events by `index` must update.

---

## Breaking Changes

### `history.log` is now `history.messages`

The active conversation getter was renamed. This is the highest-risk change
because optional-chaining patterns fail silently:

```ts
// Before
const messages = agent.history.log;

// After
const messages = agent.history.messages;
```

`agent.history.log ?? []` compiles against 0.25.x habits in untyped contexts
and silently yields `[]` on 0.26.0. Grep for `.history.log` when upgrading.

### `snapshot()` is now async

`snapshot()` is queued behind in-flight work, so a snapshot is always taken at
rest — never mid-send with dangling tool calls.

```ts
// Before
const session = agent.snapshot();

// After
const session = await agent.snapshot();
```

A non-awaited call now stores a `Promise` instead of a session object; check
every persistence call site.

### `restore()` removed — resume via the constructor

Session restore only happens at construction. The mutate-in-place `restore()`
method is gone:

```ts
// Before
const agent = new Agent(config);
agent.restore(session);

// After
const agent = new Agent(config, session);
```

`AgentSession` gains optional `archive` and `compactions` fields. Sessions
saved by earlier versions restore cleanly: a session without an `archive` has
one seeded from its restored messages, so a later compaction cannot drop
them from the record. Sessions saved by 0.26.0 carry the new fields.

### `StreamEvent` no longer carries `index`

The per-part `index` field was removed from every `stream()` event. It was a
run-global counter with no remaining consumers, and its uniqueness guarantee
had become unreliable.

- Tool and provider-tool events (`tool:request`, `tool:args-delta`,
  `tool:exec-*`, `provider-tool:*`) correlate by `id`.
- Text and thinking parts stream sequentially within a turn: deltas always
  belong to the most recently opened part.

`TurnEvent`s from `Agent.on(...)` are unaffected.

### `createHandle` export removed

`createHandle` was an internal helper. The `Handle` type remains exported.

### `TurnPart` gains a `compaction` member

`Turn` itself is unchanged — code that reads `turn.owner`, `turn.parts`, or
`turn.usage` needs no migration. Compaction (below) appears as an agent turn
containing a single `{ type: "compaction" }` part. Renderers that filter parts
by type ignore it automatically; only exhaustive `switch`es over
`TurnPart["type"]` need a new case.

## Loop Limits Are Stops, Not Errors

Hitting `maxIterations` previously returned `ok: false` with a model error of
type `"MaxIterations"`. Both limits now return a successful result carrying
everything accumulated so far:

```ts
// Before
const result = await generate({ provider, model, messages, tools, maxIterations: 5 });
if (!result.ok && result.error.kind === "model") {
  // inner error type "MaxIterations"
}

// After
const result = await generate({ provider, model, messages, tools, maxIterations: 5 });
if (result.ok && result.stopped === "max-iterations") {
  // result.messages and result.final are the complete exchange so far
}
```

`result.final.finishReason` keeps the provider's own reason for the last
message (typically `FunctionCall` — the model wanted to continue). Checks run
after each turn's tools are answered, so a stopped result always ends on a
complete exchange and can be resumed by passing `result.messages` back in.

`maxContextTokens` is new and uses the same shape with
`stopped: "token-limit"`. It compares the previous turn's reported usage
(effective input + output) against the budget.

Non-positive values for either limit now throw `INVALID_OPTIONS` at call time
instead of producing a degenerate result.

## Compaction (experimental)

Compaction is opt-in and inert unless configured. The host owns the policy
(when) and strategy (how) in one callback; the engine owns validation, receipt
stamping, state processing, and event emission.

```ts
agent.onCompaction(async ({ messages }, { usage, signal }) => {
  if (usage.total < 100_000) return null; // decline — nothing happens

  const summary = await generate({
    provider,
    model,
    signal,
    messages: [{ role: "user", content: `Summarize:\n${render(messages)}` }],
  });
  if (!summary.ok) return null;
  return [{ role: "user", content: `Summary so far: ${text(summary)}` }];
});

const record = await agent.compact(); // CompactionRecord | null
```

Key semantics:

- The returned messages become the entire active conversation. Returning
  `null` declines and leaves everything untouched.
- `history.archive` keeps the raw append-only message log; compaction never
  rewrites it. `history.compactions` records a `{ id, at }` receipt per
  compaction.
- `compact()` runs on the agent's work queue and never interleaves with an
  in-flight `send()`.
- Returned messages are validated (`COMPACTION_INVALID_MESSAGES` on dangling
  tool pairs). `validateCompactedMessages` is exported for hosts that want to
  pre-check.
- UI: `compaction:start` / `compaction:end` turn events, and an agent turn in
  `history.turns` containing a single `compaction` part (with the record once
  complete). The turn's `status` covers the lifecycle: `"streaming"` while the
  callback runs, then `"complete"` or `"error"`. Skipped compactions leave no
  turn.
- `agent.context()` returns the current `ContextUsage` estimate for deciding
  outside the callback.

The API is experimental and may change in any release.

## Upgrade Checklist

1. Rename `agent.history.log` reads to `agent.history.messages`.
2. `await agent.snapshot()` everywhere a session is persisted.
3. Replace `agent.restore(session)` with `new Agent(config, session)`.
4. Replace `error.kind === "model"` / `"MaxIterations"` handling with
   `result.ok && result.stopped === "max-iterations"`.
5. Remove any use of `StreamEvent.index`; correlate by `id` or arrival order.
6. Add a `compaction` case to any exhaustive `switch` over `TurnPart["type"]`.
7. Optionally adopt `onCompaction`/`compact()` and `maxContextTokens` for
   long-running sessions.
