# Migrating to axle 0.18.0

0.18.0 is a breaking release focused on request option naming and provider
adapter boundaries:

1. Model request controls are now first-class normalized fields.
2. Raw provider request fields now live under `providerOptions`.
3. Public request parameter types were renamed for consistency.
4. Provider adapter params now use `runtime` instead of `context`.
5. Tool choice controls are now part of the normalized model request surface.

There is no back-compat shim layer. If you are upgrading from 0.17.x, apply the
changes below.

---

## TL;DR migration

```diff
  await generate({
    provider,
    model,
    messages,
-   options: {
-     max_tokens: 500,
-     top_p: 0.9,
-     reasoning_effort: "medium",
-   },
+   maxOutputTokens: 500,
+   topP: 0.9,
+   providerOptions: {
+     reasoning_effort: "medium",
+   },
  });
```

```diff
- import type { GenerateOptions, StreamOptions } from "@fifthrevision/axle";
+ import type { GenerateParams, StreamParams } from "@fifthrevision/axle";
```

```diff
  const provider: AIProvider = {
    async createGenerationRequest(model, params) {
-     params.context.fileResolver;
+     params.runtime.fileResolver;
    },
  };
```

## Model Request Options

The old `options` bag is gone from `generate()`, `stream()`, `Agent`, and
provider request params.

### Before

```ts
await generate({
  provider,
  model,
  messages,
  options: {
    temperature: 0.2,
    top_p: 0.9,
    max_tokens: 500,
    stop: ["END"],
  },
});
```

### After

```ts
await generate({
  provider,
  model,
  messages,
  temperature: 0.2,
  topP: 0.9,
  maxOutputTokens: 500,
  stop: ["END"],
});
```

The normalized request option surface is:

```ts
interface AxleModelRequestOptions {
  reasoning?: boolean;
  maxOutputTokens?: number;
  temperature?: number;
  topP?: number;
  stop?: string | string[];
  toolChoice?: ToolChoice;
  parallelToolCalls?: boolean;
  providerOptions?: ProviderOptions;
  signal?: AbortSignal;
}
```

`frequencyPenalty`, `presencePenalty`, cache controls, provider-specific
reasoning budgets, and other non-portable fields should be passed through
`providerOptions`.

### Agent defaults and per-turn overrides

`Agent` accepts the same normalized model request options as defaults. Per-turn
options passed to `agent.send()` override those defaults.

```diff
  const agent = new Agent({
    provider,
    model,
-   options: {
-     max_tokens: 500,
-   },
+   maxOutputTokens: 500,
  });

- await agent.send("Summarize this", { options: { temperature: 0 } });
+ await agent.send("Summarize this", { temperature: 0 });
```

`providerOptions` are shallow-merged:

```ts
const agent = new Agent({
  provider,
  model,
  providerOptions: { seed: 1, metadata: { source: "agent" } },
});

await agent.send("Go", {
  providerOptions: { metadata: { source: "send" } },
});
```

The request receives:

```ts
{
  seed: 1,
  metadata: { source: "send" },
}
```

## Provider Options

Use `providerOptions` for raw provider-specific request fields.

```diff
  await generate({
    provider,
    model,
    messages,
-   options: {
-     prompt_cache_key: "thread-123",
-     prompt_cache_retention: "in_memory",
-   },
+   providerOptions: {
+     prompt_cache_key: "thread-123",
+     prompt_cache_retention: "in_memory",
+   },
  });
```

Provider adapters apply fields in this order:

1. provider defaults
2. Axle-normalized options
3. `providerOptions`

This means `providerOptions` intentionally wins if it conflicts with a
normalized Axle field.

```ts
await generate({
  provider,
  model,
  messages,
  maxOutputTokens: 500,
  providerOptions: {
    max_tokens: 1000,
  },
});
```

For a Chat Completions-compatible provider, this sends `max_tokens: 1000`.

## Type Renames

The naming scheme is now:

- `Params` means a whole argument object for a function or method.
- `Options` means a reusable optional control subset.
- `Provider*Params` means internal provider adapter params.

| Before                    | After                     |
| ------------------------- | ------------------------- |
| `AxleRequestOptions`      | `AxleModelRequestOptions` |
| `GenerateOptions`         | `GenerateParams`          |
| `StreamOptions`           | `StreamParams`            |
| `GenerateInstructOptions` | `GenerateInstructParams`  |
| `StreamInstructOptions`   | `StreamInstructParams`    |
| `GenerationRequestParams` | `ProviderGenerationParams` |
| `StreamingRequestParams`  | `ProviderStreamParams`    |
| `ProviderRequestContext`  | `ProviderRuntime`         |

Most users only need the public type renames:

```diff
  import type {
-   GenerateOptions,
-   StreamOptions,
-   GenerateInstructOptions,
-   StreamInstructOptions,
+   GenerateParams,
+   StreamParams,
+   GenerateInstructParams,
+   StreamInstructParams,
  } from "@fifthrevision/axle";
```

## Provider Runtime

Provider adapter params now use `runtime` instead of `context`.

This only affects custom `AIProvider` implementations and tests that call
provider adapters directly.

### Before

```ts
const provider: AIProvider = {
  name: "custom",
  async createGenerationRequest(model, params) {
    const tracer = params.context.tracer;
    const fileResolver = params.context.fileResolver;
  },
  async *createStreamingRequest(model, params) {
    const tracer = params.context.tracer;
  },
};
```

### After

```ts
const provider: AIProvider = {
  name: "custom",
  async createGenerationRequest(model, params) {
    const tracer = params.runtime.tracer;
    const fileResolver = params.runtime.fileResolver;
  },
  async *createStreamingRequest(model, params) {
    const tracer = params.runtime.tracer;
  },
};
```

Direct adapter tests should also change:

```diff
  await createGenerationRequest({
    client,
    model,
    messages,
-   context: {},
+   runtime: {},
  });
```

## Tool Choice Controls

Tool choice is now a normalized model request option.

```ts
await agent.send("Search the web", {
  toolChoice: { type: "tool", name: "web_search" },
  parallelToolCalls: false,
});
```

Supported values are:

```ts
type ToolChoice =
  | "auto"
  | "none"
  | "required"
  | { type: "tool"; name: string };
```

Axle maps named executable tools to provider function-tool choice syntax. Named
provider tools, such as `web_search`, are adapter-specific and may throw when a
provider cannot enforce that choice.

Unsupported normalized options fail clearly instead of being silently omitted.
For example, Gemini currently throws for `parallelToolCalls: false` because it
does not expose an equivalent guarantee.

## Provider Tools

Public `tools` and `providerTools` are still separate inputs.

```ts
const agent = new Agent({
  provider,
  model,
  tools: [calculatorTool],
  providerTools: [{ type: "provider", name: "web_search" }],
});
```

Internally, provider tools are no longer passed through the raw options bag.
Custom providers now receive them directly:

```ts
async createGenerationRequest(model, params) {
  params.providerTools;
}
```

## Cache Telemetry Checks

The cache telemetry checks now use normalized output-token controls and
`providerOptions` for provider-specific cache fields.

```diff
  await generate({
    provider,
    model,
    messages,
-   options: {
-     max_tokens: 24,
-     cache_control: { type: "ephemeral" },
-   },
+   maxOutputTokens: 24,
+   providerOptions: {
+     cache_control: { type: "ephemeral" },
+   },
  });
```

## Usage Stats

Usage stats may now include provider-reported detail fields:

```ts
type Stats = {
  in: number;
  out: number;
  cachedIn?: number;
  cacheWriteIn?: number;
  reasoningOut?: number;
};
```

`in` and `out` remain the top-line totals. Detail fields are included in those
totals when providers report them.
