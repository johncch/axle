# Migrating to Axle 0.27.0

0.27.0 replaces provider-local model enumerations with one generated model
registry. It also makes model identity explicit by using publisher-qualified
strings, such as `openai/gpt-5.5`.

This release has breaking changes for applications that import provider model
constants or library-owned default models.

---

## TL;DR

```ts
// Before
import { Anthropic, OpenAI } from "@fifthrevision/axle";

const model = OpenAI.Models.GPT_5_4_MINI;
const fallback = Anthropic.DefaultModel;

// After
import { Models } from "@fifthrevision/axle/models";

const model = Models.OpenAI.GPT_5_4_MINI;
const fallback = Models.Anthropic.CLAUDE_HAIKU_4_5;
```

Model constants now contain a publisher-qualified ID:

```ts
Models.OpenAI.GPT_5_4_MINI; // "openai/gpt-5.4-mini"
Models.Anthropic.CLAUDE_HAIKU_4_5; // "anthropic/claude-haiku-4-5"
Models.Google.GEMINI_3_5_FLASH; // "google/gemini-3.5-flash"
```

---

## One model registry

The old provider-local `models.ts` modules and their convenience exports are
removed. `OpenAI`, `Anthropic`, and `Gemini` are no longer exported objects
from `@fifthrevision/axle`.

Import model constants and metadata from the dedicated entry point:

```ts
import { ModelInfo, Models } from "@fifthrevision/axle/models";

const model = Models.OpenAI.GPT_5_5;
const limits = ModelInfo[model];

console.log(limits.contextWindow);
console.log(limits.maxOutputTokens);
console.log(limits.multimodal);
```

`Models` is organized by publisher, not by Axle transport. For example,
Qwen, Mistral, DeepSeek, MiniMax, Moonshot, and Z.AI are catalog namespaces;
`ChatCompletions` is not.

### Replace provider-local constants

| Before | After |
| --- | --- |
| `OpenAI.Models.GPT_5_4_MINI` | `Models.OpenAI.GPT_5_4_MINI` |
| `Anthropic.Models.CLAUDE_HAIKU_4_5` | `Models.Anthropic.CLAUDE_HAIKU_4_5` |
| `Gemini.Models.GEMINI_3_5_FLASH` | `Models.Google.GEMINI_3_5_FLASH` |
| `OpenAI.DefaultModel` | Choose an application-owned default explicitly |
| `Anthropic.DefaultModel` | Choose an application-owned default explicitly |
| `Gemini.DefaultModel` | Choose an application-owned default explicitly |

The old Chat Completions model export, including
`SMALL_OPEN_WEIGHT_MODELS`, is also removed. Select a concrete model from
`Models` or use the hosted provider's model ID directly.

## Publisher-qualified model strings

Registry values now include the publisher. Update any code that compares model
IDs to literal strings:

```ts
// Before
if (model === "gpt-5.5") {
  // ...
}

// After
if (model === Models.OpenAI.GPT_5_5) {
  // ...
}
```

First-party providers normalize their own qualified IDs before sending a
request to their SDK:

```ts
import { generate, openai } from "@fifthrevision/axle";

await generate({
  provider: openai(process.env.OPENAI_API_KEY!),
  model: "openai/gpt-5.5",
  messages,
});
```

The OpenAI SDK receives `gpt-5.5`. The same applies to `anthropic/...` and
`google/...` (the Gemini provider also accepts `gemini/...`). Unqualified IDs
remain accepted by first-party providers for direct use.

A qualified ID for another publisher is rejected by first-party providers:

```ts
await generate({
  provider: openai(process.env.OPENAI_API_KEY!),
  model: "zai/glm-5.2", // throws before a request is sent
  messages,
});
```

`chatCompletions()` does not normalize the model. It receives the string
unchanged, so the application remains responsible for choosing a model ID that
the configured hosted endpoint supports.

## Defaults now belong to the consumer

The core library exports no default models. Set a default in your application
where cost and capability are product decisions:

```ts
import { Models } from "@fifthrevision/axle/models";

const defaultModel = Models.OpenAI.GPT_5_4_MINI;
```

Axle CLI is a consumer and therefore has its own defaults when a first-party
job omits `provider.model`:

- OpenAI: `openai/gpt-5.4-mini`
- Anthropic: `anthropic/claude-haiku-4-5`
- Gemini: `google/gemini-3.5-flash`

Chat Completions jobs still require an explicit model.

---

## Maintaining the catalog

The registry is generated and committed to the package; applications do not
fetch Models.dev at runtime. To refresh it from Models.dev during Axle
development, run:

```sh
pnpm update-models
```

The generator keeps a rolling two-year release window and applies
publisher-specific release-date policies. Review the generated diff before
committing a refresh.
