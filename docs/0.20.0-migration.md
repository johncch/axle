# Migrating to Axle 0.20.0

0.20.0 is a breaking release focused on the core/CLI boundary.

The core runtime is now the `@fifthrevision/axle` package. The command-line
workflow runner is now the `@fifthrevision/axle-cli` package. The repo remains a
workspace so the CLI can keep dogfooding the current core runtime during
development.

There is no compatibility shim layer.

---

## Package Split

Library consumers should continue importing runtime APIs from
`@fifthrevision/axle`:

```ts
import { Agent, Instruct, anthropic } from "@fifthrevision/axle";
import { TurnAccumulator } from "@fifthrevision/axle/ui";
```

CLI users should install the CLI package:

```bash
npm install -g @fifthrevision/axle-cli
```

The `axle` binary is no longer published by the core package.

---

## Local Tools Moved Out Of Core

The core package no longer exports concrete local workflow tools.

Removed from `@fifthrevision/axle`:

- `calculatorTool`
- `execTool`
- `readFileTool`
- `writeFileTool`
- `patchFileTool`

Define application tools directly when using core:

```ts
import { Agent, type ExecutableTool } from "@fifthrevision/axle";

const tool: ExecutableTool = {
  name: "lookup",
  description: "Lookup information",
  schema,
  async execute(input) {
    return "result";
  },
};

const agent = new Agent({ provider, model, tools: [tool] });
```

CLI job files can still use these local tool names:

- `calculator`
- `exec`
- `patch-file`
- `read-file`
- `write-file`

The previous Brave search tool has been removed.

---

## Memory And Store Implementations Moved To CLI

Core keeps only the memory and store interfaces:

```ts
import type { AgentMemory, FileStore } from "@fifthrevision/axle";
```

Concrete implementations such as `ProceduralMemory` and `LocalFileStore` are no
longer part of the core runtime package. Hosts should provide their own memory
and persistence implementations, or use the CLI behavior through job files.

`AgentConfig` no longer has `scope` or `store`. `Agent` owns a stable
`sessionId`, and memory receives that id in `MemoryContext`.

---

## CLI Config Changes

The CLI no longer loads `axle.config.yml`, `axle.config.yaml`, or JSON config
files.

Provider secrets should come from the environment or a local `.env` file:

```bash
OPENAI_API_KEY=...
ANTHROPIC_API_KEY=...
GEMINI_API_KEY=...
```

Optional model overrides:

```bash
OPENAI_MODEL=gpt-4.1
ANTHROPIC_MODEL=claude-sonnet-4-5-20250929
GEMINI_MODEL=gemini-2.5-pro
```

OpenAI-compatible endpoints:

```bash
CHATCOMPLETIONS_BASE_URL=http://localhost:11434/v1
CHATCOMPLETIONS_MODEL=llama3
CHATCOMPLETIONS_API_KEY=...
```

Provider-level fields in the job file override environment defaults:

```yaml
provider:
  type: openai
  model: gpt-4.1
```

To use a non-standard environment variable, use `apiKeyEnv`:

```yaml
provider:
  type: openai
  apiKeyEnv: CUSTOM_OPENAI_KEY
```

---

## CLI Job Files

The CLI now requires an explicit YAML job file:

```bash
axle -j path/to/job.yml
```

Removed:

- default `axle.job.yml` / `axle.job.yaml` lookup
- JSON job files
- `-c, --config`

Use `.yml` or `.yaml` job files only.

---

## Agent Definition And Session APIs

0.20.0 introduces serializable agent/session types:

- `AgentDefinition`
- `AgentSession`
- `SavedAgent`
- `createAgentConfig()`
- `agent.snapshot()`
- `agent.restore(session)`

`AgentDefinition` is a serializable recipe. It is not executable by itself; the
host resolves providers and tools into an `AgentConfig`.

`AgentSession` is continuation/render state. It contains model-facing messages,
turns, session annotations, and the stable `sessionId`.

The low-level reconstruction flow is:

```ts
const config = await createAgentConfig(saved.definition, resolver);
const agent = new Agent(config);
agent.restore(saved.session);
```

Higher-level restoration ergonomics are planned for 0.21.0.
