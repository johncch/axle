# Migrating to axle 0.19.0

0.19.0 is a breaking release focused on Axle's presentation event model:

1. `AgentEvent` is now `TurnEvent`.
2. `TurnBuilder` is gone; use `TurnEventBuilder` for stream-to-turn event
   translation.
3. `TurnAccumulator` is the public reducer for `TurnEvent` streams.
4. `Turn` and `TurnPart` now support embedded annotations.
5. `Agent` now stores turns by applying `TurnEvent`s through the accumulator
   instead of mutating turn objects directly.

There is no back-compat shim layer. If you are upgrading from 0.18.x, apply the
changes below.

---

## TL;DR migration

```diff
- import type { AgentEvent, AgentEventCallback } from "@fifthrevision/axle";
+ import type { TurnEvent, TurnEventCallback } from "@fifthrevision/axle";
```

```diff
- const events: AgentEvent[] = [];
+ const events: TurnEvent[] = [];
```

```diff
- import { TurnBuilder } from "@fifthrevision/axle";
+ import { TurnEventBuilder } from "@fifthrevision/axle";
```

```diff
- function applySseEvent(event) {
-   switch (event.type) {
-     case "turn:start":
-       setTurns((turns) => [
-         ...turns,
-         { id: event.turnId, owner: "agent", parts: [], status: "streaming" },
-       ]);
-       break;
-     // many more Turn reducer cases...
-   }
- }
+ const accumulator = new TurnAccumulator<AppAnnotation, HostEvent>();
+
+ function applySseEvent(event: TurnEvent<AppAnnotation> | HostEvent) {
+   const result = accumulator.apply(event);
+   if (!result.handled) applyHostEvent(result.event);
+   setTurns(result.state.turns);
+ }
```

## Event model rename

`AgentEvent` has been renamed to `TurnEvent`.

The old name described who emitted the event. The new name describes what the
event accumulates: `TurnEvent` is the event counterpart to `Turn`.

The conceptual model is now:

```txt
StreamEvent <-> AxleMessage
TurnEvent   <-> Turn
```

`stream()` emits `StreamEvent`, which is close to provider chunks, tool
execution, and model-facing messages.

`Agent.on(...)` emits `TurnEvent`, which is organized around renderable turns,
parts, actions, and annotations.

### Type imports

```diff
  import type {
-   AgentEvent,
+   TurnEvent,
    AnnotationEvent,
    AnnotationTarget,
  } from "@fifthrevision/axle";
```

### Callback type

```diff
  import type {
-   AgentEventCallback,
+   TurnEventCallback,
  } from "@fifthrevision/axle";
```

### Event arrays and handlers

```diff
- const events: AgentEvent[] = [];
+ const events: TurnEvent[] = [];
```

```diff
- function onEvent(event: AgentEvent) {
+ function onEvent(event: TurnEvent) {
    switch (event.type) {
      case "turn:end":
        console.log(event.status);
    }
  }
```

The event string values are mostly unchanged. The type name changed because the
event layer is now explicitly the `Turn` presentation layer.

## `TurnBuilder` replaced by `TurnEventBuilder`

`TurnBuilder` has been removed.

In 0.18.x, `TurnBuilder` did two jobs:

1. translate `StreamEvent` into presentation events
2. mutate/build `Turn` objects

In 0.19.0 those jobs are split:

- `TurnEventBuilder` translates `StreamEvent -> TurnEvent`
- `TurnAccumulator` materializes `TurnEvent -> Turn[]`

### Before

```ts
const builder = new TurnBuilder();

const { turn, events } = builder.startAgentTurn();
for (const event of events) emit(event);

for (const event of builder.handleStreamEvent(streamEvent)) {
  emit(event);
}

for (const event of builder.finalizeTurn()) {
  emit(event);
}

turn.parts; // mutated by the builder
```

### After

```ts
const builder = new TurnEventBuilder();
const accumulator = new TurnAccumulator();
let turnId: string | undefined;

const start = builder.startAgentTurn();
turnId = start.turnId;
accumulator.apply(start);
emit(start);

for (const event of builder.handleStreamEvent(streamEvent)) {
  accumulator.apply(event);
  emit(event);
}

for (const event of builder.finalizeTurn()) {
  accumulator.apply(event);
  emit(event);
}

accumulator.state.turns;
const turn = accumulator.state.turns.find((candidate) => candidate.id === turnId);
```

`startAgentTurn()` now returns a single `turn:start` event, not an array.

## Turn accumulation

Use `TurnAccumulator` when you need to materialize `Turn[]` from a stream of
events.

This is especially useful for SSE/WebSocket consumers that multiplex Axle turn
events with host events such as `run:terminal`, `session:expired`, or
application-specific lifecycle events.

```ts
import { TurnAccumulator, type Annotation, type TurnEvent } from "@fifthrevision/axle";

type AppAnnotation =
  | Annotation<{ image: string }, "sandbox">
  | Annotation<{ score: number; passed: boolean }, "eval">;

type HostEvent = { type: "run:terminal"; status: string } | { type: "session:expired" };

const accumulator = new TurnAccumulator<AppAnnotation, HostEvent>();

function applyEvent(event: TurnEvent<AppAnnotation> | HostEvent) {
  const result = accumulator.apply(event);

  if (!result.handled) {
    // result.event is HostEvent here
    applyHostEvent(result.event);
  }

  render(result.state.turns);
}
```

The accumulator accepts open event streams:

- known `TurnEvent`s return `handled: true`
- unknown host events return `handled: false`
- `state.turns` contains accumulated render state
- `state.sessionAnnotations` contains session-level annotations
- turn and part annotations are embedded on their targets

The accumulator is not idempotent. If your transport can replay events, dedupe
them before applying them.

## Annotations

`Turn` and every `TurnPart` variant now accept embedded annotations.

Annotations are presentation metadata. They are useful for out-of-band UI such
as sandbox startup, eval results, deployment state, or any other consumer-owned
status that should render alongside turns without becoming model state.

```ts
type AppAnnotation =
  | Annotation<{ image: string }, "sandbox">
  | Annotation<{ score: number; passed: boolean }, "eval">;
```

```ts
const annotation: AppAnnotation = {
  id: crypto.randomUUID(),
  kind: "eval",
  label: "Plan adherence",
  placement: "after",
  status: "complete",
  data: { score: 0.92, passed: true },
};
```

Annotation events target a session, turn, or part:

```ts
const event: TurnEvent<AppAnnotation> = {
  type: "annotation:end",
  target: { type: "turn", turnId },
  annotation,
};
```

`label` is required so generic renderers have a common UI surface.

`placement` defaults to `"after"` when accumulated.

`annotation:end` defaults missing `status` to `"complete"` when accumulated.

`annotation:update` and `annotation:end` carry the full updated annotation
object. There is no patch or merge behavior.

## Turn type generics

`Turn`, `TurnPart`, and all part variants are now generic over the annotation
union supported by your renderer.

Most code does not need to specify the generic because it defaults to the base
`Annotation` type:

```ts
const turn: Turn = result.turn;
```

If your renderer wants typed annotation payloads:

```ts
type AppAnnotation =
  | Annotation<{ image: string }, "sandbox">
  | Annotation<{ score: number; passed: boolean }, "eval">;

const turns: Turn<AppAnnotation>[] = accumulator.state.turns;
```

Subagent child turns use the same annotation generic:

```ts
type Child = SubagentAction<AppAnnotation>["detail"]["children"][number];
```

## Agent turn storage

`Agent` now derives `history.turns` by applying emitted `TurnEvent`s through
`TurnAccumulator`.

This means turn objects are snapshots, not objects that are mutated in place by
`TurnBuilder`.

If you were relying on holding a reference to an old turn object and observing
it mutate during streaming, update your code to consume `TurnEvent`s or the
latest `agent.history.turns` snapshot instead.

```diff
- const turn = agent.history.latestTurn();
- // expect `turn` to mutate as streaming continues
+ agent.on((event) => {
+   // consume TurnEvent updates
+ });
+
+ const latestTurns = agent.history.turns;
```

`history.turns` still returns a copied array.

## Session restore

`session:restore` may now include session-level annotations.

```ts
const event: TurnEvent<AppAnnotation> = {
  type: "session:restore",
  turns,
  sessionAnnotations: [
    {
      id: "run-note",
      kind: "eval",
      label: "Previous eval",
      data: { score: 0.9, passed: true },
    },
  ],
};
```

Use `sessionAnnotations`, not `annotations`, for session-level annotation state.

## Removed names

The following names were removed:

| Before               | After               |
| -------------------- | ------------------- |
| `AgentEvent`         | `TurnEvent`         |
| `AgentEventCallback` | `TurnEventCallback` |
| `TurnBuilder`        | `TurnEventBuilder`  |

There are no deprecated aliases in 0.19.0.
