# Migrating to Axle 0.21.0

0.21.0 focuses on richer message formats and operational polish:

- citations on text parts
- provider reasoning/thinking details
- user-turn metadata
- simpler agent rehydration
- portable retry and timeout options

Most applications do not need large changes. The main breaking surface is for
code that renders or pattern-matches `thinking` parts.

---

## Thinking Parts Changed

`ContentPartThinking` and `ThinkingPart` now represent three provider behaviors:

- redacted thinking
- provider-supplied summaries
- renderable thinking text

Because not every provider exposes raw thinking text, `text` is now optional.
Provider continuation state moved into `continuity`.

### Before

```ts
if (part.type === "thinking") {
  renderThinking(part.text);

  if (part.encrypted) {
    saveEncryptedReasoning(part.encrypted);
  }

  if (part.signature) {
    saveSignature(part.signature);
  }
}
```

### After

```ts
if (part.type === "thinking") {
  if (part.redacted) {
    renderThinkingPlaceholder();
  } else if (part.summary) {
    renderThinkingSummary(part.summary);
  } else if (part.text) {
    renderThinkingText(part.text);
  }

  if (part.continuity?.provider === "openai") {
    saveEncryptedReasoning(part.continuity.encrypted);
  }

  if (part.continuity?.provider === "anthropic") {
    saveAnthropicContinuity(part.continuity);
  }

  if (part.continuity?.provider === "gemini") {
    saveGeminiThoughtSignature(part.continuity.thoughtSignature);
  }
}
```

Updated shape:

```ts
interface ContentPartThinking {
  type: "thinking";
  id?: string;
  text?: string;
  summary?: string;
  redacted?: boolean;
  continuity?: ThinkingContinuity;
  providerMetadata?: Record<string, unknown>;
}
```

This applies to both model-facing `AxleMessage` content and renderable `Turn`
thinking parts.

## Citations Are Attached To Text Parts

Text parts can now carry normalized provider citations.

```ts
if (part.type === "text") {
  renderText(part.text);

  for (const citation of part.citations ?? []) {
    renderCitation(citation.source);
  }
}
```

Citation shape:

```ts
interface Citation {
  source: CitationSource;
  outputSpan?: {
    start?: number;
    end?: number;
  };
  providerMetadata?: Record<string, unknown>;
}
```

`source` currently supports:

- `web`
- `document`
- `search-result`
- `retrieved-context`
- `unknown`

For example:

```ts
{
  source: {
    type: "web",
    title: "OpenAI",
    url: "https://openai.com"
  },
  outputSpan: { start: 0, end: 6 }
}
```

If you built against an unreleased 0.21 pre-release shape, update from
`provider` / `kind` / `span` to `source` / `outputSpan`.

## New Turn Events For Citations And Thinking

Consumers that handle `TurnEvent` directly should add cases for:

- `text:citation`
- `thinking:summary-delta`
- `thinking:update`

If you use `TurnAccumulator`, no extra reducer work is needed.

```ts
const accumulator = new TurnAccumulator();
const result = accumulator.apply(event);
render(result.state.turns);
```

The accumulator now updates:

- `TextPart.citations`
- `ThinkingPart.summary`
- `ThinkingPart.redacted`
- `ThinkingPart.continuity`
- `ThinkingPart.providerMetadata`

## User Turn Metadata

User messages can now carry stable host-owned metadata. Providers ignore it.
Axle stores it in history and copies it onto the corresponding user `Turn`.

```ts
await agent.send("Rewrite this prompt", {
  metadata: { surface: "prompt-editor" },
});
```

Or with `Instruct`:

```ts
const instruct = new Instruct({
  prompt: "Review this prompt",
  metadata: { surface: "prompt-review" },
});
```

Use metadata for stable facts about the message. Use annotations for mutable,
async, or explicitly placed UI state.

## Agent Rehydration Convenience

`Agent` can now restore a saved session directly from the constructor.

### Before

```ts
const agent = new Agent(config);
agent.restore(session);
```

### After

```ts
const agent = new Agent(config, session);
```

`restore(session)` still exists. The constructor form is just the shorter path.
If both `config.sessionId` and `session.sessionId` are supplied, the restored
session id wins.

## Retry And Timeout Options

Provider factories now accept portable client options:

```ts
const provider = openai(apiKey, {
  maxRetries: 2,
  timeoutMs: 30_000,
});
```

Supported by:

- `openai(apiKey, options)`
- `anthropic(apiKey, options)`
- `gemini(apiKey, options)`
- `chatCompletions(baseUrl, options)`

`chatCompletions(baseUrl, apiKey)` is still supported.

Option semantics:

- `maxRetries` is the number of retry attempts after the first request.
- `maxRetries: 0` disables retries.
- `timeoutMs` must be a positive integer.
- OpenAI, Anthropic, and Gemini map these options to SDK client options.
- Chat Completions-compatible providers use Axle's internal retry wrapper.

CLI provider config also accepts these fields:

```yaml
provider:
  type: openai
  model: gpt-5-mini
  maxRetries: 2
  timeoutMs: 30000
```

## Default Model Changes

Default models changed:

- OpenAI: `gpt-5.4-mini` -> `gpt-5-mini`
- Gemini: `gemini-3.1-flash-lite` -> `gemini-3.5-flash`

Anthropic model constants now include `CLAUDE_OPUS_4_8`.

If you need exact model stability, set `model` explicitly in code or job YAML.

## Provider Format Checks

0.21.0 adds real-provider checks for message-format behavior:

```bash
pnpm exec tsx checks/message-format/run.ts
pnpm exec tsx checks/message-format/run.ts --provider openai
pnpm exec tsx checks/message-format/run.ts --provider anthropic --case anthropic-citations
```

These checks exercise citations, thinking summaries/text, redacted thinking,
continuity payloads, and stream-to-turn shape.

There is also an internal reference for provider stream mapping:

```txt

```

## Quick Upgrade Checklist

- If you render thinking parts, handle `text?: string`, `summary`, `redacted`,
  and `continuity`.
- If you render text parts, optionally render `citations`.
- If you reduce `TurnEvent`s yourself, add cases for `text:citation`,
  `thinking:summary-delta`, and `thinking:update`.
- If you want stable render metadata on user turns, pass `metadata` to
  `agent.send()` or `Instruct`.
- If you use provider factories, consider setting `maxRetries` and `timeoutMs`
  explicitly.
- If you rely on default models, review the OpenAI and Gemini defaults.
