# Migrating to axle 0.17.0

0.17.0 is a breaking release focused on making the core response and `Instruct`
APIs more explicit:

1. Results now use an `ok: true | false` discriminant.
2. `Instruct` now takes an options object constructor.
3. `Instruct` schemas are now root Zod schemas instead of flat schema records.
4. Missing `Instruct` variables now throw a typed `InstructVariableError`.

There is no back-compat shim layer. If you are upgrading from 0.16.x, apply the
changes below.

---

## TL;DR migration

```diff
- const result = await generate({ provider, model, instruct });
- if (result.result !== "success") throw result.error;
- if (!result.response) throw new Error("Missing response");
+ const result = await generate({ provider, model, instruct });
+ if (!result.ok) throw result.error;
+ result.response; // always present when ok is true
```

```diff
- const instruct = new Instruct(
-   "Summarize {{topic}}.",
-   {
-     summary: z.string(),
-     keyPoints: z.array(z.string()),
-   },
- );
+ const instruct = new Instruct({
+   prompt: "Summarize {{topic}}.",
+   schema: z.object({
+     summary: z.string(),
+     keyPoints: z.array(z.string()),
+   }),
+ });
```

```diff
- new Instruct("Describe a {{breed}}", undefined, { vars: "optional" });
+ new Instruct({ prompt: "Describe a {{breed}}", vars: "optional" });
```

## Result shape

`generate(...)`, `stream(...).final`, and `agent.send(...).final` now resolve to
a two-state result using `ok`.

### Before

```ts
const result = await generate({ provider, model, messages });

if (result.result !== "success") {
  throw result.error;
}

if (!result.response) {
  throw new Error("Missing response");
}

result.response;
```

### After

```ts
const result = await generate({ provider, model, messages });

if (!result.ok) {
  throw result.error;
}

result.response;
```

When `ok` is `true`, `response` is always present. When `ok` is `false`, the
result has `error` and may include `final` for the raw assistant message.

The error shape is:

```ts
type GenerateError =
  | { kind: "model"; error: ModelError }
  | { kind: "tool"; error: { name: string; message: string } }
  | { kind: "parse"; error: unknown; message: string };
```

Model, tool, and parse failures return `ok: false`. Abort, fatal tool,
configuration, and unexpected execution errors still throw.

---

## `response` type by call style

The success response now depends on how you call Axle:

| Call style               | `result.response` when `ok: true`     |
| ------------------------ | ------------------------------------- |
| `generate({ messages })` | Final assistant message               |
| `stream({ messages })`   | Final assistant message               |
| `agent.send("text")`     | Assistant text                        |
| Any `Instruct` call      | Parsed schema output if schema exists |

For raw assistant content from `generate()` and `stream()`, use `result.final`.
For `Agent.send("text")`, use `result.response`.

## `Instruct` constructor

`Instruct` no longer accepts positional constructor arguments.

### Before

```ts
const instruct = new Instruct("Tell me about Mars.");
```

```ts
const instruct = new Instruct(
  "Tell me about Mars.",
  z.object({
    name: z.string(),
  }),
);
```

### After

```ts
const instruct = new Instruct({
  prompt: "Tell me about Mars.",
});
```

```ts
const instruct = new Instruct({
  prompt: "Tell me about Mars.",
  schema: z.object({
    name: z.string(),
  }),
});
```

Constructor options are:

```ts
new Instruct({
  prompt: string,
  schema?: z.ZodTypeAny,
  vars?: "required" | "optional",
});
```

`vars` defaults to `"required"`.

## Root Zod schemas

`Instruct` schemas are now root Zod schemas, usually `z.object(...)`.

### Before

```ts
const schema = {
  status: z.enum(["success", "partial", "fail"]),
  score: z.number(),
};

const instruct = new Instruct("Grade the run.", schema);
```

### After

```ts
const schema = z.object({
  status: z.enum(["success", "partial", "fail"]),
  score: z.number(),
});

const instruct = new Instruct({
  prompt: "Grade the run.",
  schema,
});
```

This removes the old flat-record constraint and allows nested schemas naturally:

```ts
const schema = z.object({
  status: z.enum(["success", "partial", "fail"]),
  metadata: z.object({
    retryable: z.boolean(),
    tags: z.array(z.string()),
  }),
});
```

Root non-object schemas are also valid:

```ts
const instruct = new Instruct({
  prompt: "Return the approval status.",
  schema: z.literal("approved"),
});
```

## Structured-output rendering

`Instruct.render()` still appends output-format instructions and a JSON example
when a schema is provided. The rendered format now targets raw JSON directly,
not JSON wrapped in XML tags.

For example:

```ts
new Instruct({
  prompt: "Review the deployment.",
  schema: z.object({
    status: z.enum(["success", "partial", "fail"]),
    retryable: z.boolean(),
  }),
}).render();
```

renders instructions like:

```md
# Output Format Instructions

Return only valid JSON matching this schema. Do not wrap it in markdown. Do not include prose before or after the JSON.

- status: "success" | "partial" | "fail"
- retryable: boolean

Example:
{
"status": "success",
"retryable": true
}

Review the deployment.
```

`z.enum(...)` and `z.literal(...)` are supported by the example renderer.

## Optional variable substitution

`Instruct` still defaults to strict variable substitution. Missing variables
throw by default.

```ts
const instruct = new Instruct({
  prompt: "Describe a {{breed}}.",
});

instruct.render(); // throws
```

To preserve unresolved placeholders, set `vars: "optional"`:

```ts
const instruct = new Instruct({
  prompt: "Describe a {{breed}}.",
  vars: "optional",
});

instruct.render(); // "Describe a {{breed}}."
```

You can also override at render time:

```ts
instruct.render({ vars: "optional" });
```

## Typed missing-variable errors

Missing `Instruct` variables now throw `InstructVariableError`, exported from the
package.

```ts
import { InstructVariableError } from "@fifthrevision/axle";

try {
  new Instruct({ prompt: "Hello {{name}}" }).render();
} catch (error) {
  if (error instanceof InstructVariableError) {
    error.code; // "INSTRUCT_VARIABLE_ERROR"
    error.missingVariables; // ["name"]
  }
}
```

The error message remains:

```txt
Missing variable: name
```

For multiple variables:

```txt
Missing variables: name, place
```

## `generate()` and `stream()` with `Instruct`

`generate()` and `stream()` both accept `instruct` as the latest user turn.
When `messages` and `instruct` are provided together, `messages` is historical
context and the rendered `Instruct` is appended as the new user message.

```ts
const result = await generate({
  provider,
  model,
  messages: previousMessages,
  instruct: new Instruct({
    prompt: "Answer {{question}}.",
    schema: z.object({
      answer: z.string(),
    }),
  }).withInput("question", "Should we proceed?"),
});

if (!result.ok) throw result.error;
result.response.answer;
```

Agent behavior is unchanged conceptually: `Agent.send(instruct)` uses the
rendered `Instruct` for one turn, then records the turn in agent history.

## Common compiler fixes

### Positional constructor no longer matches

```txt
Expected 1 arguments, but got 2.
```

Change:

```diff
- new Instruct("Answer", z.object({ answer: z.string() }))
+ new Instruct({
+   prompt: "Answer",
+   schema: z.object({ answer: z.string() }),
+ })
```

### Flat schema records no longer infer as output schema

Change:

```diff
- const schema = { answer: z.string() };
+ const schema = z.object({ answer: z.string() });
```

### `result.response` possibly missing checks are obsolete

Change:

```diff
  if (!result.ok) throw result.error;
- if (!result.response) throw new Error("Missing response");
  result.response;
```
